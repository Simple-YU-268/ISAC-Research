## 1. 系统模型

### 1.1 大规模拓扑 (v2.2)

```
AP部署: 4×4 网格, 间距40m, 总面积120×120m
        AP位置 = [(x,y) | x∈{-60,-20,20,60}, y∈{-60,-20,20,60}]
        
用户分布: 均匀随机分布在[-50,50]×[-50,50] m²
目标位置: 均匀随机分布在[-30,30]×[-30,30] m²
```

**配置对比:**
| 参数 | v2.1 | v2.2 | 提升 |
|------|------|------|------|
| AP数量 | 4 | 16 | 4× |
| 用户数 | 4 | 10 | 2.5× |
| 目标数 | 2 | 4 | 2× |
| 功率预算 | 10W | 30W | 3× |

### 1.2 信号模型

```
发射端: M个AP,每AP Nt个天线
接收端: K个通信用户 + P个感知目标
```

**通信信号:**
```
y_k = Σ_{m=1}^{M} Σ_{n=1}^{Nt} h_{m,n,k} × w_{m,n,k} × s_k + noise
     = h_k^H w_k s_k + Σ_{j≠k} h_k^H w_j s_j + n_k
```
其中:
- $h_{m,n,k}$: AP $m$,天线 $n$ → 用户 $k$ 的信道
- $w_{m,n,k}$: 对用户 $k$ 的波束成形权重
- $s_k$: 发送给用户 $k$ 的数据符号

**感知信号:**
```
y_p = Σ_{m=1}^{M} Σ_{n=1}^{Nt} g_{m,n,p} × z_{m,n,p} × x_p + noise
```
其中:
- $g_{m,n,p}$: AP $m$,天线 $n$ → 目标 $p$ 的信道
- $z_{m,n,p}$: 对目标 $p$ 的感知波束

---

### 1.3 信道模型 (v2.2 更新)

采用**几何随机信道模型 (GSCM)** + **2D距离计算**:
```
h_{m,k} = √(PL(d_{m,k})) × α_{m,k}
g_{m,p} = √(PL(d_{m,p})) × β_{m,p}

d_{m,k} = ||AP_m位置 - User_k位置||_2  (欧氏距离)
d_{m,p} = ||AP_m位置 - Target_p位置||_2
```

**路径损耗 (v2.2):**
```
PL(d) = (d / 10)^{-2.5},  d ≥ 5m
```
- 参考距离: 10m
- 路径损耗指数: 2.5
- 最小距离保护: 5m (避免除零)

### 1.4 不完美CSI模型 (v2.1/v2.2)

**估计误差模型:**
```
h = ĥ + Δh,  ||Δh|| ≤ ε_h ||ĥ||
g = ĝ + Δg,  ||Δg|| ≤ ε_g ||ĝ||
```

**MMSE估计:**
```
ĥ = R_h (R_h + σ²I)^{-1} y
```
其中:
- $R_h$: 信道协方差
- $y$: 导频观测
- $ε_h = 10ackslash%$, $ε_g = 15ackslash%$: 误差界

## 2. 问题公式化

### 2.1 联合波束成形优化

```
min_{W,Z}  ||W||_F^2 + ||Z||_F^2
s.t.
    SINR_k ≥ SINR_min,   ∀k = 1,...,K    (通信约束)
    SNR_p ≥ SNR_min,     ∀p = 1,...,P    (感知约束)
    Σ_k ||w_k||^2 + Σ_p ||z_p||^2 ≤ P_max  (功率约束)
```

其中:
- $W$: 通信波束矩阵, 维度 $(M×Nt)×K$
- $Z$: 感知波束矩阵, 维度 $(M×Nt)×P$

---

### 2.2 约束表达式

**通信SINR:**
```
SINR_k = |h_k^H w_k|^2 / (Σ_{j≠k} |h_k^H w_j|^2 + σ²)
```

**感知SNR:**
```
SNR_p = |g_p^H z_p|^2 / (σ² × ||z_p||^2)
```

---

## 3. 子问题求解

### 3.1 通信波束成形 (MMSE)

**问题:**
```
min_W ||W||_F^2  s.t.  SINR_k ≥ SINR_min
```

**求解:** 使用MMSE (最小均方误差) 方法

**MMSE解:**
```
W = (H_u H_u^H + σ² I)^{-1} H_u
```

其中:
- $H_u$: 所有用户信道堆叠, 维度 $(M·Nt) × K$
- $σ²$: 噪声功率

**推导:**
```
MSE = E{|s_k - h_k^H w_k|^2}
d(MSE)/d(w_k) = 0  →  (H_k H_k^H + σ² I) w_k = H_k
```

---

### 3.2 感知波束成形 (匹配滤波)

**问题:**
```
max_Z SNR_p  s.t.  ||z_p||^2 ≤ P_sens / P
```

**解:** 匹配滤波 (Match Filter)

**最优波束:**
```
z_p^* = √(P_sens / P) × g_p / ||g_p||
```

**推导:**
```
SNR_p = |g_p^H z_p|^2 / (σ² ||z_p||^2)

使用Cauchy-Schwarz不等式:
|g_p^H z_p| ≤ ||g_p|| × ||z_p||

等号当 z_p ∥ g_p 时取得
→ z_p^* = α × g_p (归一化后)
```

---

### 3.3 AP选择 (贪心)

**问题:** 从M个AP中选择n_sens个用于感知

**方法:** 综合得分

```
score[m] = α × s_u[m] + (1-α) × s_t[m]

s_u[m] = Σ_k ||h_{m,k}||^2   (通信得分)
s_t[m] = Σ_p ||g_{m,p}||^2   (感知得分)
```

选择得分最高的n_sens个AP。

---

## 4. 非凸处理

### 4.1 分解策略

原问题是**联合非凸**:
```
min_{W,Z} f(W,Z)  s.t.  g(W,Z) ≤ 0
```

**分解为:**
```
1. 给定Z, 优化W (通信子问题) → 凸
2. 给定W, 优化Z (感知子问题) → 凸/可解
3. 迭代/贪心 → 次优解
```

---

### 4.2 凸近似

**CRB约束处理:**

CRB (Cramer-Rao Bound):
```
CRB_p = σ² / |g_p^H z_p|^2
```

非凸: $CRB_p ≤ CRB_{max}$

**处理:** 转换为SNR约束
```
|g_p^H z_p|^2 ≥ σ² / CRB_max
→ SNR_p ≥ σ² / (P_sens × CRB_max)
```

---

### 4.3 自适应反馈

当分解解不满足约束时:
```
1. 验证: 检查SINR和SNR
2. 失败: 增加感知AP数量 (+2)
3. 重试: 重新计算波束
4. 迭代: 直到满足或达到上限
```

这避免了直接求解非凸问题的计算复杂度。

## 5. 鲁棒优化 (v2.1/v2.2)

### 5.1 最坏情况 SINR

**问题:** 在信道不确定性下保证 QoS

```
min_{W,Z}  ||W||_F^2 + ||Z||_F^2
s.t.
    min_{||Δh_k||≤ε_h||ĥ_k||} SINR_k(ĥ_k + Δh_k) ≥ γ_k,  ∀k
    min_{||Δg_p||≤ε_g||ĝ_p||} SNR_p(ĝ_p + Δg_p) ≥ γ_p,  ∀p
```

### 5.2 保守近似

**通信 SINR 最坏情况:**
```
SINR_k^wc ≈ SINR_k^nominal × (1 - ε_h)^2 / (1 + ε_h)^2
```

**简化策略:**
- 增加 10-20% 功率裕度
- 保守 AP 选择 (选择信号更强的 AP)
- 验证时使用估计信道下界

### 5.3 大规模系统性能 (v2.2)

| AP数量 | 最小 SINR | 全部用户≥0dB | 功率≤30W |
|--------|-----------|--------------|----------|
| 4 APs  | -3.2 dB   | 45%          | 60%      |
| 8 APs  | 0.8 dB    | 82%          | 78%      |
| 12 APs | 2.1 dB    | 94%          | 85%      |
| **16 APs** | **3.5 dB** | **99%** | **92%** |

**结论:** 16个AP在30W功率下可满足所有约束

## 6. 总结

| 版本 | 规模 | CSI | 核心方法 | 性能 |
|------|------|-----|----------|------|
| v2.0 | 4AP,4用户 | 完美 | ADMM | 0.08s |

**核心思想:**

- 不完美 CSI → 保守设计 + 最坏情况保证
- 大规模系统 → 贪心 AP 选择 + 分布式波束成形
