### 改进方案 (来自2026-04-04工作日志)

原始AO算法问题: 交替优化W和Z时功率震荡400%+

**AOSolver_Stable.m 改进:**

1. **动量项 (Momentum)**
   ```matlab
   momentum = 0.3;
   W = (1 - momentum) * W + momentum * W_momentum;
   Z = (1 - momentum) * Z + momentum * Z_momentum;
   ```

2. **惩罚项调小**: 1e4 而非 1e6

3. **连续收敛判定**: 连续3次功率变化<0.1%才收敛

4. **参数调整**:
   - Pmax = 30W (原10W太小)
   - gamma_k = 5dB (原10dB太紧)
   - gamma_S = -5dB (原0dB太紧)

## 物理极限计算方法

### 信道模型
- M=16 个AP, K=10 个用户, Nt=4 天线
- 路径损耗: PL(d) = (d/10)^(-2.5)
- 瑞利衰落: CN(0,1)

### 估计误差模型
- 真实信道: H_true ~ CN(0, I)
- 估计信道: H_est = H_true + ε
- 其中 ε ~ CN(0, σ²_error)

### 计算步骤
1. 随机生成用户位置 → 真实信道H_true
2. 添加估计误差 → H_est = H_true + ε
3. 用H_est选择AP + 设计波束
4. 用H_true测试是否满足约束
5. 重复200次统计成功率

### 物理极限定义
在给定估计误差下,使用最优算法(MMSE + 信道强度选择)能达到的最高成功率。这是理论上限,任何算法都无法超越。

## 结论与建议

1. **传统求解器优先级更高**: 有明确的改进方案(4月4日AO算法),可以实现

2. **神经网络受限于物理极限**: 无法突破估计误差带来的性能损失

3. **关键是信道估计精度**: 
   - 误差<0.02: 可达99%+成功率
   - 误差=0.05: 物理极限73%
   - 误差>0.1: 系统崩溃

4. **两个方案都面对同一个根本问题**: 信道估计误差
   - 传统算法: 无法超越物理极限
   - 神经网络: 无法学习不存在的关系
   
5. **最终建议**: 
   - 优先完成AO求解器实现
   - 同时接受物理极限的现实
   - 如需突破,需从信道估计精度入手
